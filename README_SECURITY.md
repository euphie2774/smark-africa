## 🚀 Quick Start (3 Steps)

### Step 1: Run Security Setup Script
```bash
venv\Scripts\python.exe setup_security.py
```

This will:
- Generate a secure SECRET_KEY
- Create a `.env` configuration file
- Set up database tables (including AuditLog)
- Create logs directory

### Step 2: Configure Environment
Edit the `.env` file created and change these critical values:
```bash
SECRET_KEY=<keep-the-generated-one>
ADMIN_PASSWORD=YourStrongPassword123!  # CHANGE THIS!
FLASK_ENV=development  # or 'production'
```

### Step 3: Add CSRF Tokens to Forms
Open each template file with a POST form and add `{{ csrf_token() }}`:

```html
<form method="POST" action="/login">
    {{ csrf_token() }}  <!-- Add this line -->
    <input type="text" name="username">
    <input type="password" name="password">
    <button type="submit">Login</button>
</form>
```

See `TEMPLATE_CSRF_EXAMPLES.md` for complete examples of all your forms.

## 💡 Key Security Points

1. **Rate Limiting**
   - Login: 5 attempts per minute
   - Registration: 3 per hour
   - Prevents brute force attacks

2. **Strong Passwords**
   - Minimum 8 characters
   - Must include: uppercase, lowercase, number, special character
   - Enforced at registration

3. **File Security**
   - No .exe, .msi, .apk files allowed
   - Path traversal protection
   - File signature verification

4. **CSRF Protection**
   - All POST requests protected
   - Tokens required in all forms
   - Prevents cross-site attacks

5. **Audit Logging**
   - All admin actions logged
   - Failed login attempts tracked
   - Stored in database and log files

## 🆘 Troubleshooting

### Problem: "CSRF token missing"
**Solution:** Add `{{ csrf_token() }}` to your form

### Problem: "SECRET_KEY must be set"
**Solution:** Run `setup_security.py` to generate one

### Problem: Can't login after changes
**Solution:** Clear browser cookies and try again

### Problem: Rate limiting too strict
**Solution:** Adjust limits in main.py:
```python
limiter = Limiter(
    default_limits=["200 per hour", "50 per minute"]  # Increase these
)
```

### Problem: File uploads not working
**Solution:** Check if file extension is in ALLOWED_EXTENSIONS
